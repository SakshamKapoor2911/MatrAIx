# Wiki Collaborator Runner Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a self-contained `./run_assignment.sh` entrypoint for worker assignment packages with first-run YAML settings, optional `uv` setup guidance, resumable runs, input integrity checks, Codex/Claude defaults, and owner-side return validation.

**Architecture:** Keep the shell script as a thin launcher and put testable behavior in `collab_kit/assignment_runner.py`. Package integrity is described by `package_manifest.json`, generated by `make_collab_package.py`, verified by the worker runner before every action, and checked again during owner merge using assignment provenance and the source SQLite database. Existing `harness.py`, `conformance.py`, and `merge_collab_results.py` remain the core run/format/merge contracts with focused extensions.

**Tech Stack:** Python standard library, Bash, JSON/JSONL, flat YAML settings implemented without PyYAML, pytest.

```bash
#!/usr/bin/env bash
set -euo pipefail
ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
RUNNER="${ROOT}/collab_kit/assignment_runner.py"

if [[ ! -f "${RUNNER}" ]]; then
  echo "ERROR: missing ${RUNNER}" >&2
  exit 2
fi

if command -v uv >/dev/null 2>&1; then
  exec uv run --no-project python "${RUNNER}" "$@"
fi
if command -v python3 >/dev/null 2>&1; then
  exec python3 "${RUNNER}" "$@"
fi
if command -v python >/dev/null 2>&1; then
  exec python "${RUNNER}" "$@"
fi

echo "ERROR: Python 3 is required. Install python3 or uv, then rerun ./run_assignment.sh." >&2
exit 2
```
