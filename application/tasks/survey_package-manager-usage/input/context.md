# Package Manager Usage

How developers choose and use package managers across JavaScript, Python, system, and other language ecosystems.

## About this survey

This survey profiles package-manager usage, opening with a screener on frequency, tenure, role, and primary language ecosystem. It captures tooling through multi-selects covering JavaScript, Python, system, and other-language package managers, plus single-choice items on the primary JavaScript and Python tools, lockfile handling, security handling, update cadence, automated update bots, and a workspace-tools multi-select. Attitudes are measured with an agreement matrix about the respondent's primary package manager and an importance matrix over selection criteria, along with varied Likert items on audit frequency, switching to newer tools, supply-chain concern, and ecosystem fragmentation, a top-three factor ranking, and a pain-point multi-select. Later sections capture satisfaction with the primary tool and install speed, a 0–10 recommendation scale, and trial and switching intent, with numeric free-text on dependency counts and tools used per week and an optional open comment. The survey closes with a demographic block covering role, professional experience, employment, company size, industry, age, gender, race and ethnicity, education, U.S. region, and urbanicity.
