# CSS Framework Usage

How developers choose and use CSS frameworks, methodologies, and component libraries to style user interfaces.

## About this survey

This survey profiles CSS styling practices, opening with a screener on frequency, tenure, and the respondent's relationship to styling. It records the primary CSS approach and uses multi-selects to capture frameworks and UI libraries, methodologies, preprocessors, CSS-in-JS libraries, component libraries, and build tools, plus single-choice items on responsive design, theming, dark mode, and CSS resets. Attitudes are measured with an agreement matrix about the respondent's primary approach and an importance matrix over selection criteria, along with varied Likert items on utility-first CSS, CSS-in-JS complexity, component-library preference, and custom-CSS frequency, a top-three factor ranking, and a challenges multi-select. Later sections capture satisfaction with the approach and tooling, a 0–10 recommendation scale, and adoption and switching intent, with numeric free-text on project counts and years of experience and an optional open comment. The survey closes with a demographic block covering role, professional experience, employment, company size, industry, age, gender, race and ethnicity, education, U.S. region, and urbanicity.
