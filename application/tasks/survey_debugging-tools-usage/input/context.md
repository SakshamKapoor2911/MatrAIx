# Debugging Tools Usage

How developers debug software, including the tools, techniques, and habits they rely on.

## About this survey

This survey profiles how developers debug code, opening with a screener on frequency, tenure, role, and primary debugging method. It records the debuggers and dev tools used, debugging techniques, languages debugged, logging and error-tracking tools, the print-versus-debugger preference, common bug types, and tools reserved for hard problems. Attitudes are captured with an agreement Likert battery on print debugging, debugger value, confidence, and frustration, and a per-item importance matrix over what matters when choosing a debugging approach, plus frequency scales for irreproducible bugs and production debugging. Additional sections cover production-debugging approach, editor debuggers, pain points, first steps on a new bug, a top-three ranking of techniques, AI-assisted debugging, satisfaction with tools and skills, recommendation likelihood, and learning intent. Question types include single-choice, multi-select, Likert scales, an importance matrix, a ranking item, and numeric free-text. The survey concludes with a demographic block covering role, professional experience, employment, company size, industry, age, gender, race and ethnicity, education, U.S. region, and urbanicity.
