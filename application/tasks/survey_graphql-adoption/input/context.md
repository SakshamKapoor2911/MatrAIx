# GraphQL Adoption

How developers adopt, use, and evaluate GraphQL relative to REST and other API styles.

## About this survey

This survey examines GraphQL adoption, beginning with a screener on usage status, tenure, frequency, and the respondent's relationship to GraphQL. It records the dominant API style and captures tooling through multi-selects on client libraries, server implementations, languages, exploration IDEs, federation and gateway tools, and use cases, along with single-choice items on caching, schema-design approach, the REST-to-GraphQL mix, and subscriptions. Attitudes are measured with an agreement matrix comparing GraphQL to REST and an importance matrix over API-selection criteria, plus varied Likert items on complexity, greenfield choice, security concerns, and performance frequency, a top-three adoption-reason ranking, and a challenges multi-select. Later sections cover satisfaction with tooling and performance, a 0–10 recommendation scale, and expansion and abandonment intent, with numeric free-text on GraphQL API counts and share of APIs and an optional open comment. The survey ends with a demographic block covering role, professional experience, employment, company size, industry, age, gender, race and ethnicity, education, U.S. region, and urbanicity.
