# JavaScript Runtime Preference

How developers choose, use, and evaluate JavaScript runtimes such as Node.js, Deno, and Bun.

## About this survey

This survey profiles JavaScript runtime preferences, beginning with a screener on frequency, tenure, and role. It records the primary runtime and captures tooling through multi-selects covering runtimes used, version managers, package managers, frameworks, deployment platforms, and workloads, along with single-choice items on TypeScript usage, how TypeScript is run, and adoption status for Bun and Deno. Attitudes are measured with an agreement matrix about the respondent's primary runtime and an importance matrix over runtime-selection criteria, plus varied Likert items on feature-adoption frequency, Node's dominance, Bun's production readiness, and Deno's security model, a top-three reason ranking, and a pain-point multi-select. Later sections capture satisfaction with the runtime and its performance, a 0–10 recommendation scale, and trial and switching intent, with numeric free-text on service counts and years of experience and an optional open comment. The survey closes with a demographic block covering role, professional experience, employment, company size, industry, age, gender, race and ethnicity, education, U.S. region, and urbanicity.
